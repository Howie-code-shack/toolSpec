# Example Custom Rule

This is an example rule file. Rules provide specific instructions or constraints for Claude when working on your project.

## When to Use Rules

Create rule files for:
- Project-specific coding standards
- Security requirements
- Performance guidelines
- Domain-specific constraints
- Workflow requirements

## Example Rule

```
When writing API endpoints:
1. Always validate input parameters
2. Use proper HTTP status codes
3. Include error handling for all external calls
4. Log all requests with relevant context
5. Follow RESTful conventions
```

## Naming Convention

Name your rule files descriptively:
- `security-guidelines.md`
- `api-standards.md`
- `error-handling.md`
- `commit-message-format.md`

Delete this example file and create your own rules as needed.
